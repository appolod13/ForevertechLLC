## Contribution Guidelines

### Coding Standards
- TypeScript: strict mode, no `any` unless justified
- Follow ESLint and Prettier rules; CI will enforce
- Keep functions small and focused

### Commit Style
- Conventional commits: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`
- Include scope when relevant: `feat(api): ...`

### Tests
- Add unit tests for new logic
- Keep coverage above baseline; prefer small, focused tests

### Review
- Provide context and rationale in PR descriptions
- Link related issues or tasks
